Domande Preliminari

  • Devo dotare la parte Java di una GUI, come fare? Operare nel backend basandosi su Model View Control. Il Model dovrebbe essere la parte di software Java su cui abbiamo già lavorato.
    Per View si intende la visualizzazione del Model (pagina html), ma fungera anche da input (campiare InputMock). Il control in cosa consiste? Dovrei interporre il LifeController tra Model e View.
  • Tramite richiesta http, il server restituirebbe una nuova pagina modificata. Posso evitare che la richiesta venga eseguita senza creare ogni volta una nuova pagina? Tramite WebSocket è possibile.
  • Se uso AJAX potrei facilmente impostare l'interfaccia MachineToMachine a partire dalla HMI? In cosa consiste questo passaggio?
    Devo fare in modo che l'interfaccia comunichi con il software java tramite comunicazione Machine to Machine. Così facendo potrei automatizzare i test tramite stessa tecnica.
    Il mio obiettivo rimane creare un microservizio.
  • Non rimane che concentrarmi sul controller lato SpringBoot

Problem analysis

Architettura

Modello Client-Server: Utilizzare un'architettura basata su client-server, dove il server ospita la logica di gioco (ConwayLife25) e gestisce le connessioni degli utenti. La GUI sarà implementata come un'applicazione web che comunica con il server tramite WebSocket.

Interfaccia Grafica

Design: Interfaccia utente semplice e intuitiva che consenta agli utenti di interagire facilmente con la simulazione, utilizzando rappresentazioni diverse per celle vive e morte, mostrando lo stato dell'esecuzione (se le celle sono tutte morte non è detto che l'utente abbia terminato l'esecuzione. Controlli: Includere pulsanti per avviare, fermare e resettare la simulazione. Assicurarsi che ci sia un feedback visivo chiaro quando le azioni vengono eseguite. Griglia: Utilizzare una rappresentazione grafica della griglia delle celle, con la possibilità di interagire con le celle per impostare il loro stato iniziale. Implementare una griglia dinamica che si aggiorna in tempo reale.

Comunicazione

WebSocket: Utilizzare WebSocket per la comunicazione tra il client e il server, ciò consente aggiornamenti in tempo reale della griglia. Owner: Per la comunicazione, bisogna inviare comandi dal client al server solo quando l'owner interagisce con la simulazione, nonostante tutti gli utenti connessi vedano l'evoluzione della griglia.

Framework e Tecnologie

SpringBoot: Utilizzare SpringBoot per gestire la logica del server, semplificando la creazione di applicazioni web senza alterare lo sviluppo Java.

Caso d'uso

  • Un utente apre la pagina HTML e stabilisce una connessione (tramite WebSocket) con il server
  • L'owner imposta lo stato iniziale delle celle e avvia la simulazione
  • Il server calcola la nuova generazione di celle e modifica solo le celle che cambiano stato a tutti gli utenti connessi
  • Gli utenti possono continuare a osservare l'evoluzione del gioco in tempo reale e interagire con la griglia (se utente owner) interrompendola, riavviandola, resettando la griglia o continuando a modificare le celle
